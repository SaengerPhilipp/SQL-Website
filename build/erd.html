ERD

General ERD:

This is an ERD(entity-relationship-diagram) in the form you can create them with pgadmin 4. Public is just the folder in which the ERD is saved on the postgreSQL server. Account is the name of the table. The rest are all columns and ID is the primary key. Next to the name of the columns is the data type that can be stored in the column.

1:N Connection:

The line on the site of the table indicates that it is connected to another table. A connection works via a foreign key. That means one of the tables has to have a column that represents the primary key of the other table. Those are called 1:n connections because in this case in the table with the foreign key there can only be one primary key of the other table that is placed into one of the rows, but this one primary key could be in more than one row. So one row can only contain one primary key of the other table and therefore can only connect to one row of the other table. However the same primary key could be in infinite foreign keys therefore one row of the table without the foreign key could connect to an infinite amount of rows of the other table.

N:N Connection:

If you want to connect infinite rows of one table to infinite rows of another table and the other way around you need to create a N:N connection. In order to do that it is not enough to create an foreign key in one of the tables you actually you need to create a third table with foreign keys of both the other tables. However that means you can also save information about this connection in this third table.

Atomisation:

Data that is in a database table should always be atomised because you could have problems analysing your data otherwise. That means in one column there should be only one information that you can not split up into multiple smaller informations.

Redundancy:

You should try to avoid Redundancies in a database table as well because it can lead to database anomalies. For example, you are more likely to get a deletion or update anomaly because you have to change data in a lot of rows rather than in one. That means if multiple columns are connected that means that they depend on each other then they should be extracted in another table. After that, they can be connected to the other table via a 1:N connection.
