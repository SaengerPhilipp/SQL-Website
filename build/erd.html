ERD

General ERD:

This is an ERD(entity-relationship-diagram) in the form you can create them with pgadmin 4. Public is just the folder in which the ERD is saved on the postgreSQL server. Account is the name of the table. The rest are all columns and ID is the primary key. Next to the name of the columns is the data type that can be stored in the column. The line on the site of the table indicates that it is connected to another table. A connection works via a foreign key. That means one of the tables has to have a column that represents the primary key of the other table. Those are called 1:n connections because in this case in the table with the foreign key there can only be one primary key of the other table that is placed into one of the columns, but this one primary key could be in more than one columns. So one columns can only contain one primary key of the other table and therefore can only connect to one column of the other table. However the same primary key could be in infinite foreign keys therefore one columns of the table without the foreign key could connect to an infinite amount of columns of the other table. If you want to have an

SELECT columnCondition
FROM table alias
WHERE condition
ORDER BY orderColumn direction ;

OrderColumn:

The variable orderColumn equals alias.column which defines the column according to whose values the table is ordered.

Direction:

The variable direction can have either the value ASC or DESC depending on whether you like to order the table ascending or descending.

Example:

In this example we have a database table called accounts with the columns id, name, surname, birthday, email, phoneNumber and password. We want to display every account where te name starts with an A and te person is born befor the year 2000. We also want to order the table alphabetically descending by the column name. In order to do this we write the following command:

SELECT *
FROM accounts a
WHERE a.name LIKE 'A%'
AND a.birthday < '2000-01-01'
ORDER BY a.name DESC;
