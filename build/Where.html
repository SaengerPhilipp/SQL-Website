Conditions

General command:

SELECT columnCondition
FROM table alias
WHERE condition ;

Condition:

A condition would look like this: instead of condition we would write alias.column = 'value'. In this case value defines the value an object has to have in the defined column to be displayed.

Multiple conditions:

If you want that an object has to match two or more conditions you connect thse conditios with an AND like this:

SELECT columnCondition
FROM table alias
WHERE condition
AND condition2 ;

If you want that an object has to match one out of two or more conditions you connect thse conditios with an OR like this:

SELECT columnCondition
FROM table alias
WHERE condition
OR condition2 ;

Special conditions:

Instead of = you can write <, >, <=, >= and != to use greater than, smaller than, greater than or equal to, smaller than or equal, and not equal to to create a condition. Dates are in the format YYYY-MM-DD so you can also use them in conditions with greater than or smaller than comparisons. If one wants a comparison where something should be similar to something but not the same instead of = one uses LIKE or NOT LIKE and than you replace the unnecessary parts of the value with a "%".

Example:

In this example we have a database table called accounts with the columns id, name, surname, birthday, email, phoneNumber and password. We want to display every account where te name starts with an A and te person is born befor the year 2000. In order to do this we write the following command:

SELECT *
FROM accounts a
WHERE a.name LIKE 'A%'
AND a.birthday < '2000-01-01';
